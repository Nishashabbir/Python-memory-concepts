: What are Python Modules (Core Idea)

A module is simply a file containing Python code that you can reuse.

 Instead of writing everything in one file, you split code into multiple files and import them.

Example

Create a file:

math_utils.py
def add(a, b):
    return a + b

Now in another file:

import math_utils

print(math_utils.add(2, 3))  # 5

 You just used a module.

STEP 2: __name__ == "__main__" (VERY IMPORTANT)

This is not just syntax — it controls how your module behaves.

 Core Idea

Every Python file has a built-in variable:

__name__

Its value depends on how the file is used.

 Case 1: Run file directly
# file: test.py

print(__name__)

Output:

__main__

 When you run a file directly → Python sets __name__ = "__main__"

 Case 2: Import the file
# file: test.py
print(__name__)
# file: main.py
import test

Output:

test

 When imported → __name__ = module_name

 Why This Matters

Sometimes you want:

Some code to run only when file is executed directly
NOT when imported
 Correct Pattern
def greet():
    print("Hello!")

if __name__ == "__main__":
    print("Running directly")
    greet()
Behavior:
Situation	Output
Run file directly	prints everything
Import file	ONLY functions available, no execution
 Without This (Common Mistake)
# bad_module.py
print("This runs always!")

Now:

import bad_module

 Output:

This runs always!

 This is BAD in real projects

 Real Use Cases
Testing code
Debugging
Writing reusable modules
CLI tools
 Interview-Level Understanding

If interviewer asks:

 “Why use if __name__ == "__main__"?”

Strong answer:

It ensures that certain parts of code run only when the file is executed directly, not when it's imported as a module. This helps separate reusable logic from execution logic.


Using the dir() Function
There is a built-in function to list all the function names (or variable names) in a module. The dir() function:

Example
List all the defined names belonging to the platform module:

import platform

print(x)
Note: The dir() function can be used on all modules, also the ones you create yourself.


How Python Handles Imports Internally

Most people use modules. Few understand what actually happens.

 1. Modules are loaded ONLY ONCE (Caching)

When you import a module, Python:

Loads it into memory
Stores it in:
import sys
print(sys.modules)

 This is a dictionary of already loaded modules

 Example
my_module.py
print("Module is loading...")
main.py
import my_module
import my_module

 Output:

Module is loading...

 Only runs once

 Why?

Because Python caches it in:

sys.modules["my_module"]

So second import just reuses it.

 Interview Trap
 Question: “What happens if you import the same module multiple times?”

Strong answer:

Python loads it once and reuses it from sys.modules, avoiding re-execution.

 2. Forcing Reload (Advanced)

Sometimes you WANT to reload (rare but important)

 Clean + safe

Problem:

 Fix 1 (Quick Fix)

Move import inside function:

Circular Import (Execution Flow) /////////////////////////////////
# a.py
import b

# b.py
import a
Step-by-step:
Python starts loading a
a imports b
Python starts loading b
b tries importing a again
But a is not fully initialized yet

 Result:

incomplete module
missing attributes
errors / unexpected behavior
